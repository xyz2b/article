# 发布物料说明

本目录产出了一套可分发资源，用于把《一个 7 行的 C 函数，是怎么一路变成 CPU 上的电信号的》这篇文章配上交互演示。
公众号/知乎正文里不能跑 JS，所以采用「正文放静态配图 + 外链按钮跳到独立交互页」的方案。

## 文件清单

| 文件 | 用途 |
|---|---|
| `c-to-cpu.html` | **独立可分发的交互版**。直接托管或发链接，手机/电脑均可单步、自动播放。这就是要放外链的页面。 |
| `c-to-cpu-visualizer.html` | 桌面交互版（早期版本，逻辑相同，无介绍页/移动适配），本地自用。 |
| `figs/frame_*.png` | 6 张关键帧配图（2x 高清），直接贴进正文。 |

## 一、托管交互页（拿到外链）

`c-to-cpu.html` 是纯单文件，零依赖，随便挑一个：

- **GitHub Pages**：把 `c-to-cpu.html` 推到仓库，开 Pages，链接形如 `https://<用户名>.github.io/<仓库>/c-to-cpu.html`
- **Cloudflare Pages / Vercel / Netlify**：拖一个文件夹上去即得链接
- **腾讯云 COS / 七牛 / 任意静态托管**：上传后取公网 URL

> 公众号外链限制：订阅号正文不能直接放任意可点击链接，常见做法是放在「阅读原文」、或引导到自己已认证的服务/小程序。知乎可直接在正文插超链接。

## 二、正文配图（贴 figs/ 里的 PNG）

建议按文章章节顺序插入，每张配一句图注：

| 配图 | 建议插入位置 | 图注 |
|---|---|---|
| `frame_04_main_frame.png` | 第二节「栈帧」 | main 序言跑完：RBP 钉死当基准线，RSP 挖出局部变量空间 |
| `frame_07_callq.png` | 第四节「call」 | `callq` 一瞬：返回地址 0x3e 压栈，RIP 飞进 add |
| `frame_13_add.png` | 第一节 / 第五节「加法」 | 那个 `+` 的真身：`addl` 机器码 `03 45 f8`，eax=3+4=7，a/b/c 写在红区 |
| `frame_17_ret.png` | 第五节「ret 拆帧」 | `retq` 弹回返回地址，add 的帧整块作废 |
| `frame_19_store.png` | 第五节「收返回值」 | 从信箱 %eax 取出 7 存进 x，`int x = add(3,4)` 完成 |
| `frame_23_done.png` | 第七节「连起来」 | 程序结束，栈面归零——一压一弹，干干净净 |

## 四、重新生成配图（如改了页面）

```bash
cd /Users/xyzjiao/CareerPlanning
CHROME="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
FILE="file://$PWD/c-to-cpu.html"
for pair in "04_main_frame:4" "07_callq:7" "13_add:13" "17_ret:17" "19_store:19" "23_done:23"; do
  name="${pair%%:*}"; step="${pair##*:}"
  "$CHROME" --headless --disable-gpu --hide-scrollbars \
    --force-device-scale-factor=2 --window-size=1280,1100 \
    --screenshot="figs/frame_${name}.png" "${FILE}?shot=1&step=${step}"
done
```

页面支持 URL 参数：`?step=N` 跳到第 N 步（0–23），`?shot=1` 隐藏控件与介绍便于截图。
